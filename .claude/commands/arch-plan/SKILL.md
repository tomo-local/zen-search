# Arch Plan

`/arch-analyze` で洗い出した問題について、複数の対応方針を検討してトレードオフを整理し、推奨案を選定します。
分析ファイルに「対応方針」セクションを追記します。

## 実行手順

### Step 1: 入力の確認

引数 `$ARGUMENTS` を確認してください：

- **`.analysis/` ファイルパス** → そのファイルから問題一覧を読み込む
- **問題の説明（テキスト）** → 引数をそのまま問題として扱う
- **引数なし** → `.analysis/` 内の最新ファイルを使用する

`.analysis/` ファイルが指定された場合、ファイルを読み込んで問題一覧と問題詳細を把握してください。

### Step 2: 関連コードの読み込み

各問題の「該当ファイル」に記載されたコードを実際に読み込み、現状の実装を把握してください。
改善案を考える上で必要なファイルがあれば追加で読み込んでください。

### Step 3: 問題ごとに対応方針を複数検討

各問題に対して **2〜3つの対応方針（Option A / B / C）** を検討してください。

各オプションには以下を含めること：

**記載項目**:

- 概要: 何をするか（1〜2行）
- 実装イメージ: コードスニペット（変更前/後）
- メリット: この方針で得られるもの
- デメリット: 失うもの・増えるコスト
- 影響範囲: 変更が及ぶファイル・依存箇所

**検討の観点**:

- 最小変更で済む保守的な修正 vs 根本解決する大きなリファクタ
- 型安全性の向上度合い
- 既存テスト・ストーリーへの影響
- 段階的にリリースできるか

### Step 4: 推奨案の選定

各問題について推奨オプションを1つ選び、選定理由を明記してください。

選定基準:

- リスクと効果のバランス
- コードベースの現状に対する実現可能性
- 他の問題の修正と組み合わせやすいか

### Step 5: 分析ファイルへの追記

`.analysis/` ファイルの各問題詳細に「対応方針」セクションを追記してください。

**追記フォーマット**:

````markdown
#### 対応方針

| オプション          | 概要 | メリット | デメリット |
| ------------------- | ---- | -------- | ---------- |
| **Option A**        | ...  | ...      | ...        |
| **Option B**        | ...  | ...      | ...        |
| **Option C** (任意) | ...  | ...      | ...        |

**推奨**: Option X — [選定理由を1〜2行で]

<details>
<summary>Option A: 実装イメージ</summary>

```ts
// Before
...

// After
...
```
````

</details>

<details>
<summary>Option B: 実装イメージ</summary>

```ts
// Before
...

// After
...
```

</details>
```

### Step 6: 結果のサマリー

以下の形式でユーザーに報告してください：

```
## 対応方針の検討結果

| # | 問題 | 深刻度 | 推奨オプション |
|---|------|--------|----------------|
| 1 | ... | 重大/中/低 | Option X |

`.analysis/YYYY-MM-DD-<対象名>-review.md` に対応方針を追記しました。

次のステップ:
- Issueを作成する → `/arch-issue .analysis/YYYY-MM-DD-<対象名>-review.md`
```
